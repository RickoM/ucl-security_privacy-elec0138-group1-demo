UE — User Equipment
Simulated mobile phone. Holds USIM with subscriber credentials: SUPI (permanent identity), K (128-bit secret key), OPc (operator key). Conceals identity as SUCI using ECIES encryption before transmission over the air.
gNB — Next Generation Node B
Simulated 5G base station. Relays NAS messages between UE and AMF over the N2 interface (NGAP protocol). Handles radio resource management and forwards user plane data to UPF via GTP-U tunnels (N3).
AMF — Access and Mobility Management Function
Entry point for UE registration. Manages UE mobility, NAS security, and authentication orchestration. Calls AUSF via N12 to initiate 5G-AKA. Assigns 5G-GUTI as temporary identity after successful registration.
AUSF — Authentication Server Function
Runs 5G-AKA authentication procedure. Receives SUCI from AMF, calls UDM via N13 to get authentication vectors, stores XRES*, verifies RES* from UE, releases KSEAF on success. Rejects if RES* ≠ XRES*.
UDM/ARPF — Unified Data Management
Decrypts SUCI to recover SUPI. Runs the real Milenage algorithm (f1–f5) using K and OPc stored in MongoDB. Generates RAND, AUTN, XRES*, CK, IK and KAUSF. K never leaves UDM — this is the root of 5G security.
UPF — User Plane Function
5G data gateway. Forwards user plane packets between gNB (N3/GTP-U) and the internet (N6/SGi). Creates the ogstun TUN interface on EC2. All data traffic from the UE passes through UPF on its way to the internet.

⚠ Attack 3 — SQN Desynchronisation DoS Repeated SYNC_FAILURE overwhelms UDM · Legitimate UE blocked · TS 33.102 §6.3.5
How SQN DoS works
■ Attacker registers normally — gets RAND + AUTN challenge from AMF
■ Instead of RES*, sends SYNC_FAILURE with forged AUTS token
■ Goes through AMF → AUSF → UDM as a legitimate 3GPP resync
■ Repeated rapidly — UDM overwhelmed processing resyncs
■ Legitimate UE registration denied — DoS achieved
⚠ Simulated Attack — Unlike Attacks 1 & 2 which hit real Open5GS endpoints, this attack is simulated in the FastAPI proxy. A real SQN DoS requires a rogue UERANSIM gNB sending crafted AUTS tokens directly to Open5GS — complex to set up safely in a demo. The threat vector is real and documented in TS 33.102 §6.3.5.

🛡 WAF Mitigation — Rate Limiting blocks SQN DoS WAF intercepts after 3 attempts · HTTP 429 · UDM protected
● WAF OFF
WAF Rule
■ Allow first 3 SYNC_FAILURE requests (legitimate threshold)
■ Block all subsequent requests → HTTP 429 Too Many Requests
■ Open5GS UDM never receives the attack traffic
■ Legitimate UE registration unaffected
⚠ Simulated Attack · ✓ Real Mitigation — The SQN DoS attack is simulated in the FastAPI proxy (same as Stage 6). However the WAF rate limiter is real application-layer code — it genuinely intercepts requests and returns HTTP 429. Toggle WAF OFF → all 8 pass through. Toggle WAF ON → only 3 allowed, rest blocked. This demonstrates the same rate-limiting principle used in production 5G WAF deployments (TS 33.117).
